Implementation Approach

How to build the Neural Network hub world with max wow factor

A

Three.js + Custom Shaders (Recommended)

Core: Three.js for 3D scene, custom GLSL shaders for neon glow/energy effects, vanilla JS for game controls.

Pros: Battle-tested, huge ecosystem, shader flexibility for acid neon effects, works on all browsers, tons of examples to reference.
Cons: WebGL-based (not cutting-edge WebGPU), but visual result is identical for this scope.
Wow factor: Custom bloom shaders, particle systems, post-processing (bloom, chromatic aberration, film grain). This is how 99% of award-winning WebGL sites are built.

B

WebGPU + Three.js Hybrid

Core: Three.js WebGPU renderer (experimental) with TSL (Three Shading Language) for compute-powered particle effects.

Pros: Cutting edge tech, GPU compute for massive particle counts, shows you're on the bleeding edge.
Cons: WebGPU support is Chrome/Edge only (~70% of users). Needs WebGL fallback. TSL is less documented. More bugs.
Wow factor: Same visual result but can handle 10x more particles via compute shaders. Impressive to technical reviewers.

C

WebGPU Native (No Three.js)

Core: Raw WebGPU API with WGSL shaders. Custom render pipeline from scratch.

Pros: Maximum flex — shows deep graphics programming knowledge. Full control over every pixel.
Cons: 10x more code, no ecosystem, Chrome-only, months of work for same visual result. High risk of scope creep.
Wow factor: Technically impressive to GPU engineers but visually same. Overkill for a portfolio.

💡 Recommendation: Option A (Three.js + Custom Shaders) gives you 95% of the wow factor with 20% of the risk. We can still mention WebGPU awareness in the site's "tech used" section. The visual result is what creates wow — not the API underneath.
